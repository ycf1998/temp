现有的那三个 skiill 是 DIP 提供的（对我们有用的应该就两个），不知道我们的目录和配置约定：api-spec/wsplus 和 config.yaml；
因此我们需要单独写一个 skill，直接把它们的 SKILL.md 作为我们 skill 里 reference/ 下的参考文件；

这个 skill 核心是如下两件事（根据指令看是要做哪一个）：
1. 生成 wplus 接口的 yaml：输入主要是两种，一个是 API 设计文档，一个是代码（Controller），然后调用 mcp 生成 yaml 放到我们约定的目录；
   如果返回的 yaml 和 文档/代码不一致，需要告知用户，由用户处理；如果是已有的接口，即已有 yaml ，那调用 mcp 的时候要使用 上一次 yaml 的那个参数；
2. 发布 wplus 接口：需要做一个检查，对于所有要发布的接口，仔细比对 yaml 和代码，保证完全一致（url、method、入出参、字段名、字段类型、层级等都必须完全一样，不一致则不允许发布）；
   发布的话，他需要一个网关配置，config.yaml 里有默认的，将其加入到要发布接口的 yaml；然后调用 mcp 发布网关配置和发布 wplus；
